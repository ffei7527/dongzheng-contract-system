# 🚀 新手完整教程：上传到 GitHub 并运行网站

本教程将一步步教你如何将项目上传到 GitHub 并让网站运行起来。

---

## 📋 第一步：准备工作

### 1.1 创建 GitHub 账户

1. 打开浏览器，访问：https://github.com
2. 点击右上角的 **Sign up**（注册）
3. 填写信息：
   - 用户名（Username）- 这个会显示在你的网站地址中
   - 邮箱地址
   - 密码
4. 完成验证，创建账户

### 1.2 安装 Git（如果还没有）

**检查是否已安装 Git：**
1. 按 `Win + R` 键
2. 输入 `cmd` 按回车
3. 在黑色窗口中输入：`git --version`
4. 如果显示版本号，说明已安装，跳到下一步
5. 如果显示"不是内部或外部命令"，需要安装

**安装 Git：**
1. 访问：
2. 下载 Git for Windows
3. 双击安装程序
4. 一直点击"下一步"，使用默认设置
5. 安装完成后重启电脑

---

## 📦 第二步：在 GitHub 上创建仓库

### 2.1 创建新仓库

1. 登录 GitHub
2. 点击右上角的 **+** 号，选择 **New repository**（新建仓库）
3. 填写信息：
   - **Repository name**（仓库名称）：输入 `contract-demo` 或 `dongzheng-contract-system`
   - **Description**（描述）：可选，输入"东证融汇合同系统"
   - **Public**（公开）或 **Private**（私有）：选择 Public
   - **不要**勾选 "Add a README file"（我们已经有了）
   - **不要**勾选 "Add .gitignore"（我们已经有了）
4. 点击 **Create repository**（创建仓库）

### 2.2 记住仓库地址

创建成功后，你会看到一个页面，上面有仓库地址，类似：
```
https://github.com/你的用户名/contract-demo.git
```
**把这个地址复制下来，后面要用！**

---

## 💻 第三步：上传代码到 GitHub

### 3.1 打开命令提示符

**方法 1：**
1. 按 `Win + R` 键
2. 输入 `cmd` 按回车

**方法 2：**
1. 在文件夹地址栏输入 `cmd` 按回车
2. 或者右键点击文件夹，选择"在此处打开命令窗口"

### 3.2 进入项目文件夹

在命令提示符中输入（复制粘贴，然后按回车）：

```bash
cd "C:\Users\Administrator\Desktop\合约应用程序"
```

### 3.3 配置 Git 用户信息（重要！）

**必须先配置，否则会报错！**

依次输入以下命令（替换成你的信息）：

```bash
git config --global user.name "你的名字"
```

```bash
git config --global user.email "你的邮箱"
```

例如：
```bash
git config --global user.name "张三"
git config --global user.email "zhangsan@example.com"
```

**或者更简单：双击运行 `修复Git配置.bat` 文件，会自动配置！**

### 3.4 初始化 Git 仓库

依次输入以下命令（每输入一行按回车，等待完成后再输入下一行）：

```bash
git init
```

```bash
git add .
```

```bash
git commit -m "首次提交：东证融汇合同系统"
```

### 3.5 连接到 GitHub 仓库

**⚠️ 重要提示：**
1. **命令必须以 `git` 开头！** 不能只写 `remote add origin`
2. **仓库地址只能有一个！** 不要复制多个地址
3. **地址格式要正确！** 末尾是 `.git`，中间没有空格

**正确格式：**
```bash
git remote add origin https://github.com/你的用户名/仓库名.git
```

**错误示例（不要这样做）：**
- ❌ `remote add origin ...` （缺少 git）
- ❌ `git remote add origin 地址1地址2` （两个地址连在一起）
- ❌ `git remote add origin 地址 .git` （地址末尾有空格）

**例如，如果你的用户名是 `zhangsan`，仓库名是 `contract-demo`，那么命令应该是：**
```bash
git remote add origin https://github.com/zhangsan/contract-demo.git
```

**如果提示 "remote origin already exists"（远程仓库已存在），先删除再添加：**
```bash
git remote remove origin
git remote add origin https://github.com/你的用户名/仓库名.git
```

```bash
git branch -M main
```

```bash
git push -u origin main
```

**如果这是第一次连接，可能会提示输入用户名和密码。**

**注意：** 执行 `git push` 时，可能会要求输入用户名和密码：
- **用户名**：你的 GitHub 用户名
- **密码**：不是你的 GitHub 密码，而是 **Personal Access Token**（个人访问令牌）

### 3.5 创建 Personal Access Token（如果需要）

如果提示需要密码，按以下步骤创建：

1. 登录 GitHub
2. 点击右上角头像 > **Settings**（设置）
3. 左侧菜单最下方，点击 **Developer settings**
4. 点击 **Personal access tokens** > **Tokens (classic)**
5. 点击 **Generate new token** > **Generate new token (classic)**
6. 填写信息：
   - **Note**：输入 `我的电脑` 或任意名称
   - **Expiration**：选择过期时间（建议 90 天）
   - **勾选** `repo`（全部权限）
7. 点击 **Generate token**
8. **重要：** 复制生成的 token（只显示一次，要保存好！）
9. 在命令提示符中粘贴这个 token 作为密码

---

## 🌐 第四步：启用 GitHub Pages

### 4.1 等待 GitHub Actions 完成

1. 上传代码后，进入你的 GitHub 仓库页面
2. 点击上方的 **Actions**（操作）标签
3. 你会看到 "Deploy to GitHub Pages" 正在运行
4. 等待 1-2 分钟，直到看到绿色的 ✓ 号，表示成功

### 4.2 配置 GitHub Pages

1. 在仓库页面，点击 **Settings**（设置）
2. 左侧菜单找到 **Pages**（页面）
3. 在 **Source**（源）下拉菜单中，选择 **GitHub Actions**
4. 如果已经显示 "Your site is published at..."，说明已经成功！

### 4.3 访问你的网站

部署成功后，你的网站地址是：
```
https://你的用户名.github.io/仓库名/
```

例如：
```
https://zhangsan.github.io/contract-demo/
```

**恭喜！你的网站已经上线了！** 🎉

## ❓ 常见问题

### Q1: 命令提示符显示"不是内部或外部命令"
**A:** 说明 Git 没有安装或没有添加到系统路径。重新安装 Git，安装时选择"添加到 PATH"。

### Q1.5: 显示 "fatal: unable to auto-detect email address"
**A:** 需要先配置 Git 用户名和邮箱。运行 `修复Git配置.bat` 或手动输入：
```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

### Q1.6: 显示 "remote不是内部或外部命令"
**A:** 命令缺少 `git` 前缀！正确写法是 `git remote add origin ...`，不是 `remote add origin ...`

### Q2: git push 时提示"Authentication failed"
**A:** 需要使用 Personal Access Token 而不是密码。按照上面的步骤创建 token。

### Q3: 网站显示 404 或空白
**A:** 
- 等待几分钟，GitHub Pages 需要时间部署
- 检查 GitHub Actions 是否成功（绿色 ✓）
- 清除浏览器缓存，按 `Ctrl + F5` 强制刷新

### Q4: 找不到 Settings 或 Pages 选项
**A:** 确保你登录的是仓库的拥有者账户，不是访客。

### Q5: GitHub Actions 显示失败
**A:** 
- 点击失败的 Action 查看错误信息
- 检查 `package.json` 文件是否存在
- 确保所有文件都已上传
