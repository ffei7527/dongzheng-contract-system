# 查看构建错误 - 详细步骤

## 如何查看具体的构建错误

### 步骤 1：打开失败的运行

1. 进入 GitHub 仓库：https://github.com/ffei7527/dongzheng-contract-system
2. 点击 **Actions** 标签
3. 点击最新的失败运行（红色 ✗）

### 步骤 2：查看构建步骤的详细日志

1. 在运行详情页面，找到 **build** 任务（左侧）
2. 点击 **build** 任务
3. 展开 **Build** 步骤
4. **复制所有错误信息**，特别是：
   - 红色的错误消息
   - 堆栈跟踪（Stack trace）
   - 任何 "Error:" 或 "Failed" 开头的消息

### 步骤 3：常见错误及解决方法

#### 错误 1：找不到模块（Cannot find module）
**解决方法：**
- 检查 `package.json` 中的依赖是否正确
- 确保所有导入的文件都存在

#### 错误 2：TypeScript 类型错误
**解决方法：**
- 检查 `tsconfig.json` 配置
- 可能需要修复代码中的类型错误

#### 错误 3：Tailwind CSS 错误
**解决方法：**
- 检查 `tailwind.config.js` 配置
- 检查 `postcss.config.js` 配置
- 确保 `index.css` 文件存在

#### 错误 4：Vite 构建错误
**解决方法：**
- 检查 `vite.config.ts` 配置
- 查看具体的错误信息
