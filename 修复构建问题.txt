   修复构建问题 - Artifacts 太小（只有 2.44 KB）

【问题】Artifacts 只有 2.44 KB，说明构建不完整

【已修复】已更新工作流配置，添加了调试步骤

【下一步操作】
─────────────────────────────────────────────────────────
1. 提交更改：
   git add .
   git commit -m "修复：添加构建调试步骤"
   git push

2. 等待 GitHub Actions 自动运行

3. 查看构建日志：
   - 进入 Actions 标签
   - 点击最新的运行
   - 展开 "List dist contents" 步骤
   - 查看 dist 文件夹的内容和文件大小

4. 如果 dist 文件夹为空或文件很小：
   - 检查 "Build" 步骤的日志
   - 查看是否有错误信息
   - 告诉我具体的错误

【正常情况】
─────────────────────────────────────────────────────────
正常的构建产物应该：
- dist/index.html（几 KB）
- dist/assets/index-xxx.js（几百 KB 到几 MB）
- dist/assets/manifest-xxx.json（几 KB）
- 总大小应该至少 100 KB 以上

如果 Artifacts 只有 2.44 KB，说明：
- 构建可能失败了
- 或者只上传了部分文件
- 需要检查构建日志
