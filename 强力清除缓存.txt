   强力清除缓存 - 解决演示版本问题
═══════════════════════════════════════════════════════════

【方法 1】使用开发者工具（最有效！）
─────────────────────────────────────────────────────────
1. 按 F12 打开开发者工具
2. 右键点击浏览器的刷新按钮
3. 选择"清空缓存并硬性重新加载"
4. 或者：
   - 在开发者工具中点击 Network（网络）标签
   - 勾选 "Disable cache"（禁用缓存）
   - 保持开发者工具打开
   - 按 Ctrl + F5 刷新

【方法 2】无痕模式测试
─────────────────────────────────────────────────────────
1. 按 Ctrl + Shift + N 打开无痕窗口
2. 访问：https://ffei7527.github.io/dongzheng-contract-system/
3. 如果无痕模式显示完整版本，说明是缓存问题

【方法 3】清除所有浏览器数据
─────────────────────────────────────────────────────────
1. 按 Ctrl + Shift + Delete
2. 时间范围：选择"全部时间"
3. 勾选：
   ✓ 浏览记录
   ✓ Cookie 和其他网站数据
   ✓ 缓存的图像和文件
4. 点击"清除数据"
5. 关闭所有浏览器窗口
6. 重新打开浏览器

【方法 4】验证部署内容
─────────────────────────────────────────────────────────
1. 访问：https://ffei7527.github.io/dongzheng-contract-system/assets/index-BLBVQid4.js
   - 如果能看到代码，说明已部署
   - 如果 404，说明部署有问题

2. 访问：https://ffei7527.github.io/dongzheng-contract-system/index.html
   - 右键 → 查看网页源代码
   - 应该看到引用 ./assets/index-BLBVQid4.js
   - 不应该看到"演示版本"

【方法 5】强制重新部署
─────────────────────────────────────────────────────────
1. GitHub → Actions → "Deploy to GitHub Pages"
2. 点击 "Run workflow" → 选择 master → Run workflow
3. 等待 2-3 分钟
4. 清除缓存后访问
